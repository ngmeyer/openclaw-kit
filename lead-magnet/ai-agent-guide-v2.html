Security & Privacy

‚ö†Ô∏è Don't Expose Your Instance to the Internet

OpenClaw runs locally by default. Keep it that way unless you know what you're doing. If you must expose it (for remote access), use SSH tunneling or VPN‚Äînever direct port forwarding without authentication.

üîê API Keys Stay Local

Your OpenRouter key, email credentials, Telegram tokens‚Äîthey're stored on your machine, not in the cloud. This is a feature, not a bug. Just back them up securely.

üõ°Ô∏è The Privacy Trade-Off

Yes, when you use Claude or GPT-4 via OpenRouter, your prompts still go to Anthropic/OpenAI. But your context (memory files, past conversations, personal data) stays local. You're trading "zero external API calls" for "access to frontier models." If you need 100% local, run Llama or Mistral locally‚Äîbut you'll sacrifice capability.

Model Selection Mistakes

‚ùå Don't Use Opus for Everything

I burned $200 in my first month using Opus as my default model. It's amazing, but overkill for 80% of tasks. Use Sonnet. Save Opus for when you're truly stuck.

‚úÖ DO Mix Models in the Same Session

OpenClaw lets you switch models mid-conversation. Use Haiku to triage your inbox, then switch to Sonnet when you find something that needs a thoughtful response. Game-changing for cost control.

Workflow Design

üîÑ Automate the Boring Stuff

If you do it every day, automate it with a cron job. Morning briefings, email triage, calendar sync‚Äîset it once, forget it forever. This is where the magic happens.

üìù Teach Your Agent About You

Edit SOUL.md and USER.md. Tell it your preferences, work style, communication tone. The more context you give, the better it gets. My agent knows I prefer bullet points over paragraphs, that I'm Pacific time, and that I hate jargon. Yours should too.

üß† Memory Management is Key

Your agent maintains MEMORY.md (long-term knowledge) and daily logs. Review them weekly. Prune outdated info. Add lessons learned. Think of it like tending a garden‚Äîa little maintenance yields huge returns.
